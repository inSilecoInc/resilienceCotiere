5.1 Abondance (A)

L’abondance est un indicateur qui permet d’évaluer le nombre total d’individus appartenant à différentes espèces dans une unité d’échantillonnage donnée (Barnes et al. 1991). Contrairement à la richesse spécifique, qui se concentre uniquement sur la diversité des espèces, l’abondance mesure le total des individus recensés, indépendamment du nombre d’espèces présentes.

L’abondance totale par événement est calculée comme suit :

\[ A_i = \sum_{s=1}^{S} X_{i,s} \tag{5.1}\]

où :

  • \(A_i\) = Nombre total d’individus recensés lors de l’événement \(i\),
  • \(S\) = Nombre total d’espèces présentes dans le jeu de données,
  • \(X_{i,s}\) = Nombre total d’individus de l’espèce \(s\) lors de l’événement \(i\).

Les étapes spécifiques de l’évaluation de l’indicateur sont les suivantes :

  1. Filtrage des données d’abondance
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Deux sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
  2. Agrégation par événement
    • Pour chaque événement d’échantillonnage (event_id), la somme des individus recensés pour toutes les espèces est calculée.
  3. Conversion en format matriciel
    • Une transformation des données est réalisée afin d’obtenir une table où :
      • Les événements sont en lignes.
      • Les espèces sont en colonnes.
      • Les cellules contiennent l’abondance totale par espèce.
  4. Calcul de l’indicateur d’abondance séparément pour l’abondance (n) et la densité (n/m²) à partir de Equation 5.1.
  5. Export des résultats
    • Les résultats sont sauvegardés sous format .csv dans le fichier abondance.csv.

5.2 Biomasse (B)

L’indice de biomasse permet d’évaluer la quantité totale de matière vivante dans un échantillon donné (Barnes et al. 1991) et est souvent exprimée en grammes de matière par unité de surface (\(g/m^2\)). Dans notre contexte, nous avons utilisé des données de biomasse standardisées en g/m² pour assurer la comparabilité entre échantillons. Ceci implique que les données de biomasse rapportées en \(g\) uniquement n’ont pas été considérées pour le calcul de cet indicateur. L’indicateur de biomasse pour chaque événement est la somme de la biomasse de toutes les espèces présentes :

\[ B_i = \sum_{s=1}^{S} X_{i,s} \tag{5.2}\]

où :

  • \(B_i\) est la biomasse totale lors de l’événement \(i\).
  • \(S\) est le nombre total d’espèces.
  • \(X_{i,s}\) est la biomasse mesurée pour l’espèce \(s\) lors de l’événement \(i\).

Les étapes spécifiques sont les suivantes :

  1. Filtrage des données de biomasse:
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Deux sous-ensembles sont créés :
      • Données en g : Biomasse totale.
      • Données en n/m² : Biomasse totale par unité de surface.
  2. Agrégation par événement et espèce: pour chaque événement d’échantillonnage (event_id), la biomasse totale par espèce (species_id) a été calculée.
  3. Conversion en format matriciel :
    Une transformation des données a été réalisée afin d’obtenir une table où :
    • Les événements sont en lignes.
    • Les espèces sont en colonnes.
    • Les cellules contiennent la biomasse totale par espèce.
  4. Calcul de l’indicateur d’abondance séparément pour la biomasse (g) et la biomasse par unité de surface (g/m²) à partir de Equation 5.2.
  5. Export des résultats: les valeurs calculées sont exportées sous format .csv au sein du fichier biomasse.csv.

5.3 Dominance (DOM)

L’indice de dominance permet d’évaluer la répartition des espèces au sein d’un échantillon et de quantifier la mesure dans laquelle certaines espèces dominent la communauté (Barnes et al. 1991). Une valeur élevée indique qu’une ou quelques espèces sont très dominantes, tandis qu’une valeur plus faible traduit une répartition plus équilibrée des abondances entre espèces. Cet indicateur est basé sur l’indice de dominance de Simpson, qui est défini comme suit :

\[ D_i = \sum_{s=1}^{S} p_{i,s}^2 \tag{5.3}\]

où :

  • \(D_i\) est l’indice de dominance pour l’événement \(i\).
  • \(S\) est le nombre total d’espèces dans l’événement \(i\).
  • \(p_{i,s}\) est l’abondance relative de l’espèce \(s\) dans l’événement \(i\), définie comme :

\[ p_{i,s} = \frac{X_{i,s}}{\sum_{s=1}^{S} X_{i,s}} \]

avec \(X_{i,s}\) étant l’abondance de l’espèce \(s\) lors de l’événement \(i\).

L’indice varie entre 0 et 1, où 1 indique une dominance totale d’une seule espèce et 0 une distribution égale entre les espèces.

Les étapes spécifiques sont les suivantes :

  1. Filtrage des données d’abondance
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Deux sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
  2. Agrégation par événement et espèce :
    Pour chaque événement d’échantillonnage (event_id), l’abondance totale par espèce (species_id) a été calculée.
  3. Conversion en format matriciel :
    Une transformation des données a été réalisée afin d’obtenir une table où :
    • Les événements sont en lignes.
    • Les espèces sont en colonnes.
    • Les cellules contiennent l’abondance totale par espèce.
  4. Calcul des abondances relatives :
    L’abondance de chaque espèce a été divisée par l’abondance totale de l’événement pour obtenir sa proportion relative.
  5. Calcul de l’indice de dominance :
    L’indice de Simpson a été calculé en utilisant vegan::diversity(index = "simpson") du package R vegan (Oksanen et al., 2024).
  6. Export des résultats: les valeurs calculées sont exportées sous format .csv au sein du fichier dominance.csv.

5.4 Richesse Spécifique (S)

L’indice de richesse spécifique (S) représente le nombre total d’espèces présentes dans un échantillon donné (Peet, 1974). La richesse spécifique considère uniquement la présence d’espèces, sans pondération par leur quantité. L’indicateur de richesse spécifique pour chaque événement est calculé comme suit :

\[ S_i = \sum_{s=1}^{S} I(X_{i,s} > 0) \tag{5.4}\]

où :

  • \(S_i\) est le nombre total d’espèces recensées lors de l’événement \(i\).
  • \(S\) est le nombre total d’espèces dans l’ensemble des données.
  • \(X_{i,s}\) est l’abondance mesurée de l’espèce \(s\) lors de l’événement \(i\).
  • \(I(X_{i,s} > 0)\) est une fonction indicatrice prenant la valeur 1 si l’espèce est présente et 0 sinon.

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données d’abondance
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Deux sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
  2. Agrégation par événement et espèce : Pour chaque événement d’échantillonnage (event_id), la somme des abondances par espèce (species_id) a été calculée.
  3. Conversion en format matriciel : Une transformation des données a été réalisée afin d’obtenir une table où :
    • Les événements sont en lignes.
    • Les espèces sont en colonnes.
    • Les cellules contiennent l’abondance totale par espèce.
  4. Calcul de la richesse spécifique : L’indicateur de richesse spécifique a été calculé en appliquant vegan::specnumber() du package R vegan (Oksanen et al., 2024), qui compte le nombre total d’espèces présentes par événement.
  5. Export des résultats : Les valeurs calculées sont exportées sous format .csv (richesse_specifique.csv).

5.5 Diversité de Shannon-Wiener (H’)

L’indice de Shannon-Wiener (H’) est une mesure de la diversité des espèces qui prend en compte la richesse spécifique et l’équitabilité des abondances (Gray et al. 1982). Contrairement aux indices purement quantitatifs, il pondère la présence d’espèces par leur abondance relative, offrant ainsi une vision plus équilibrée de la structure des communautés. L’indice est défini comme suit :

\[ H'_i = -\sum_{s=1}^{S_i} p_{i,s} \ln(p_{i,s}) \tag{5.5}\]

où :

  • \(H'_i\) est l’indice de diversité de Shannon-Wiener pour l’événement \(i\).

  • \(S_i\) est le nombre total d’espèces dans l’échantillon \(i\).

  • \(p_{i,s}\) est l’abondance relative de l’espèce \(s\) :

    \[ p_{i,s} = \frac{X_{i,s}}{\sum_{s=1}^{S_i} X_{i,s}} \]

  • \(X_{i,s}\) est l’abondance de l’espèce \(s\) lors de l’événement \(i\).

L’indice de Shannon varie généralement entre 0 et 5 :

  • Valeurs faibles (\(H' \approx 0\)) : Peu d’espèces dominantes, faible diversité.
  • Valeurs élevées (\(H' \gg 2\)) : Répartition plus homogène des espèces.

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données d’abondance

    • Deux sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
  2. Agrégation par événement et espèce
    • Pour chaque événement d’échantillonnage (event_id), la somme des abondances par espèce (species_id) a été calculée.
  3. Conversion en format matriciel
    • Une transformation des données a été réalisée afin d’obtenir une table où :
      • Les événements sont en lignes.
      • Les espèces sont en colonnes.
      • Les cellules contiennent l’abondance totale par espèce.
  4. Calcul de l’indice de Shannon (H')
    • L’indice a été calculé en appliquant vegan::diversity(index = "shannon") du package R vegan (Oksanen et al., 2024).
  5. Export des résultats
    • Les valeurs calculées sont exportées sous format .csv (diversite_shannon.csv).

5.6 Équitabilité de Piélou (J’)

L’indice d’équitabilité de Piélou (J’) est une mesure de l’uniformité de la répartition des individus entre les espèces d’un échantillon (Piélou, 1966). Il permet d’évaluer dans quelle mesure les espèces coexistent de manière équilibrée ou si certaines dominent fortement la communauté. Une valeur élevée indique une distribution équitable des abondances, tandis qu’une valeur faible signale la prédominance de quelques espèces sur les autres.

Cet indice est calculé à partir de l’indice de diversité de Shannon (H’) et du nombre total d’espèces (S) dans l’échantillon :

\[ J'_i = \frac{H'_i}{\ln(S_i)} \tag{5.6}\]

où :

  • \(J'_i\) est l’indice d’équitabilité de Piélou pour l’événement \(i\).

  • \(H'_i\) est l’indice de diversité de Shannon-Wiener (voir Section 5.5).

  • \(S_i\) est le nombre total d’espèces dans l’événement \(i\).

  • \(p_{i,s}\) est l’abondance relative de l’espèce \(s\) dans l’échantillon :

    \[ p_{i,s} = \frac{X_{i,s}}{\sum_{s=1}^{S_i} X_{i,s}} \]

L’indice de Piélou varie entre 0 et 1 :

  • \(J' \approx 1\) → Toutes les espèces ont une abondance similaire (équitabilité élevée).
  • \(J' \approx 0\) → Une ou quelques espèces dominent fortement (équitabilité faible).

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données d’abondance
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Deux sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
  2. Agrégation par événement et espèce
    • Pour chaque événement d’échantillonnage (event_id), la somme des abondances par espèce (species_id) a été calculée.
  3. Conversion en format matriciel
    • Une transformation des données a été réalisée afin d’obtenir une table où :
      • Les événements sont en lignes.
      • Les espèces sont en colonnes.
      • Les cellules contiennent l’abondance totale par espèce.
  4. Calcul de l’indice de Shannon (H')
    • L’indice de Shannon a été calculé en appliquant vegan::diversity(index = "shannon").
  5. Calcul de la richesse spécifique (S)
    • La richesse spécifique a été déterminée avec vegan::specnumber(), qui compte le nombre total d’espèces présentes par événement.
  6. Calcul de l’indice d’équitabilité de Piélou (J')

5.7 Diversité de Simpson (λ)

L’indice de Simpson (λ) est une mesure de la dominance qui reflète la probabilité que deux individus sélectionnés au hasard dans un échantillon appartiennent à la même espèce (Simpson, 1949). Contrairement à l’indice de Shannon qui mesure l’information contenue dans une communauté, l’indice de Simpson met davantage l’accent sur les espèces dominantes.

L’indice est défini comme suit :

\[ \lambda_i = \sum_{s=1}^{S_i} p_{i,s}^2 \tag{5.7}\]

où :

  • $ _i $ est l’indice de Simpson pour l’événement \(i\).

  • $ S_i $ est le nombre total d’espèces dans l’échantillon \(i\).

  • $ p_{i,s} $ est l’abondance relative de l’espèce \(s\), définie comme :

    \[p_{i,s} = \frac{X_{i,s}}{\sum_{s=1}^{S_i} X_{i,s}}\]

  • $ X_{i,s} $ est l’abondance de l’espèce \(s\) lors de l’événement \(i\).

L’indice varie entre 0 et 1 :

  • Valeurs faibles (\(\lambda \approx 0\)) : Haute diversité, la dominance est répartie sur plusieurs espèces.
  • Valeurs élevées (\(\lambda \approx 1\)) : Une espèce est dominante, indiquant une faible diversité.

      • Données en n : Nombre total d’individus.

    • Pour chaque événement d’échantillonnage (event_id), la somme des abondances par espèce (species_id) a été calculée.

      • Les événements sont en lignes.

      • Les cellules contiennent l’abondance totale par espèce.
  4. Calcul de l’indice de Simpson (λ)

  5. Export des résultats

5.8 Série d’Indices de Hill (HILL)

La série d’indices de Hill est une famille de mesures de diversité permettant d’examiner la structure des communautés biologiques sous différents angles (Hill, 1973). Ces indices prennent en compte à la fois la richesse spécifique, l’équitabilité et la dominance selon une pondération contrôlée par un paramètre \(q\).

L’indice général de Hill est défini comme :

\[ {}^qD = \left( \sum_{s=1}^{S} p_s^q \right)^{\frac{1}{1-q}} \tag{5.8}\]

où :

  • \({}^qD\) est l’indice de diversité de Hill d’ordre \(q\).
  • \(S\) est le nombre total d’espèces dans l’échantillon.
  • \(p_s\) est l’abondance relative de l’espèce \(s\).
  • \(q\) est un paramètre qui contrôle l’importance des espèces dominantes :
    • \(q = 0\) → Nombre total d’espèces (richesse spécifique).
    • \(q = 1\) → Indice de Shannon exponentiel (pondération équilibrée).
    • \(q = 2\) → Inverse de l’indice de Simpson (favorise les espèces dominantes).

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données d’abondance
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Deux sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
  2. Agrégation par événement et espèce
    • Pour chaque événement d’échantillonnage (event_id), la somme des abondances par espèce (species_id) a été calculée.
  3. Conversion en format matriciel
    • Une transformation des données a été réalisée afin d’obtenir une table où :
      • Les événements sont en lignes.
      • Les espèces sont en colonnes.
      • Les cellules contiennent l’abondance totale par espèce.
  4. Calcul des indices de Hill
    • \(D_0\) : Richesse spécifique (vegan::specnumber())
    • \(D_1\) : Exponentielle de l’indice de Shannon (exp(vegan::diversity("shannon")))
    • \(D_2\) : Inverse de l’indice de Simpson (1 / vegan::diversity("invsimpson"))
  5. Export des résultats
    • Les valeurs calculées sont exportées sous format .csv (indices_hill.csv).

5.9 Raréfaction de Hurlbert (HURB)

Vérifier valeurs d’abondance

Il faudra vérifier les données d’abondance puisque certaines semblent avoir des valeurs après la décimales, bien qu’il s’agisse de décompte d’individus. Pour l’instant, les données d’abondance sont arrondies afin de permettre d’appliquer la méthode. Il faudra cependant vérifier les données afin de valider leurs valeurs.

La raréfaction de Hurlbert est une approche permettant de standardiser la richesse spécifique en fonction de la taille de l’échantillon, corrigeant ainsi les biais dus aux différences d’effort d’échantillonnage. Elle est particulièrement utile pour comparer des communautés échantillonnées à des intensités variables.

L’indicateur est défini comme suit :

\[E(S_n) = S - \sum_{s=1}^{S} \left( 1 - \frac{{N - X_s}}{{N}} \right)^n\]

où :

  • \(E(S_n)\) est le nombre d’espèces attendu pour un échantillon de taille \(n\).
  • \(S\) est le nombre total d’espèces observées dans l’échantillon.
  • \(X_s\) est l’abondance de l’espèce \(s\).
  • \(N\) est le nombre total d’individus dans l’échantillon.
  • \(n\) est le seuil d’échantillonnage choisi.

L’idée principale est de simuler un processus de sous-échantillonnage et d’estimer combien d’espèces seraient observées si tous les échantillons avaient la même taille.

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données d’abondance
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Deux sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
  2. Agrégation par événement et espèce
    • Pour chaque événement d’échantillonnage (event_id), la somme des abondances par espèce (species_id) a été calculée.
  3. Conversion en format matriciel
    • Une transformation des données a été réalisée afin d’obtenir une table où :
      • Les événements sont en lignes.
      • Les espèces sont en colonnes.
      • Les cellules contiennent l’abondance totale par espèce.
  4. Application de la raréfaction de Hurlbert
    • L’indicateur est calculé en appliquant vegan::rarefy(data, sample = n), où \(n\) est la taille d’échantillon standardisée.
  5. Export des résultats
    • Les valeurs calculées sont exportées sous format .csv (rarefaction_hurlbert.csv).

5.10 Diversité Taxonomique (TAXD)

Consolidation de la taxonomie

Comme mentionné à la Section 4.1.4, plus de travail est nécessaire afin d’obtenir une table taxonomique convenable. Les résultats de cette analyse devraient ainsi être revisités une fois la liste taxonomique convenablement consilidée.

NA distance taxonomique

Des NA sont présents au sein de l’analyse de distance taxonomique effectuée avec la fonction vegan::taxa2dist. Pour l’instant, les distances manquantes sont remplacées par une valeur de 1, représentant une distance maximale. Cette supposition devra toutefois être vérifée si les NA persistent une fois la consolidation de la taxonomie effectuée.

L’indice de diversité taxonomique, développé par Warwick & Clarke (1995, 1998), évalue la biodiversité en tenant compte des relations évolutives entre espèces plutôt que de se baser uniquement sur leur nombre ou leur abondance. Il repose sur les distances taxonomiques entre espèces d’une communauté pour fournir une mesure plus robuste de la diversité biologique. Contrairement aux indices classiques (ex. Shannon, Simpson), cet indice considère l’éloignement taxonomique entre espèces au sein d’un échantillon. Ainsi, une communauté composée d’espèces réparties dans plusieurs phylums sera considérée plus diverse qu’une autre contenant uniquement des espèces du même genre.

L’analyse de diversité taxonomique est effectuée grâce à la fonction vegan::taxondive du package R vegan (Oksanen et al., 2024). Cette dernière retourne six métriques permettant d’évaluer la structure taxonomique d’une communauté :

Métrique Description Équation Interprétation
D Somme des distances taxonomiques entre toutes les paires d’espèces dans l’échantillon \(D = \sum_{i<j} \omega_{ij}\) Reflète la diversité taxonomique totale, indépendamment du nombre d’espèces. Un D élevé indique une forte séparation taxonomique.
Dstar Version ajustée de \(D\) pour le nombre d’espèces \(D^* = \frac{D}{S-1}\) Corrige l’effet du nombre d’espèces sur \(D\). Permet de comparer des échantillons avec des richesses spécifiques différentes.
Lambda Variation totale de la diversité taxonomique dans l’échantillon \(\Lambda = \sum_{i<j} (\omega_{ij} - \overline{\omega})^2\) Mesure la dispersion taxonomique. Un Λ élevé signifie une forte variabilité taxonomique dans l’échantillon.
Δ+ (Dplus) Distinctivité Taxonomique Moyenne (moyenne des distances taxonomiques entre les espèces présentes) \(\Delta^+ = \frac{\sum_{i<j} \omega_{ij}}{S(S-1)/2}\) Plus Δ+ est élevé, plus les espèces présentes sont éloignées taxonomiquement, ce qui indique une communauté diversifiée.
sd.Δ+ (sd.Dplus) Écart-type de Δ+ (dispersion des distances taxonomiques) \(sd(\Delta^+) = \sqrt{\frac{\sum_{i<j} (\omega_{ij} - \Delta^+)^2}{S(S-1)/2}}\) Mesure la variabilité des distances taxonomiques. Une forte dispersion indique une distribution inégale des taxons.
Λ+ (SDplus) Variation de la Distinctivité Taxonomique \(\Lambda^+ = \frac{\sum_{i<j} (\omega_{ij} - \Delta^+)^2}{S(S-1)/2}\) Un Λ+ faible signifie que les distances taxonomiques sont relativement homogènes. Un Λ+ élevé traduit une répartition déséquilibrée des taxons.

Les étapes spécifiques de l’analyse de diversité taxonomique sont les suivantes :

  1. Extraction des espèces présentes
    • Les indices sont calculés séparément sur :
      • Les données d’abondance brute (n).
      • Les données de densité (n/m²).
  2. Récupération des informations taxonomiques
    • Chaque espèce est associée à sa hiérarchie taxonomique (Règne → Phylum → Classe → Ordre → Famille → Genre).
    • Ces niveaux sont utilisés pour calculer les distances taxonomiques entre espèces.
  3. Construction de la matrice de distance taxonomique
    • vegan::taxa2dist() est utilisé pour créer une matrice de distances taxonomiques entre espèces.
    • La distance entre deux espèces est basée sur leur éloignement taxonomique :
      • Deux espèces appartenant à un même genre auront une distance faible.
      • Deux espèces appartenant à des phylums différents auront une distance élevée.
  4. Transformation des occurrences en matrice présence/absence
    • Pour chaque type de données, une table de présence/absence ou d’abondance est construite où :
      • Les événements sont en lignes.
      • Les espèces sont en colonnes.
      • Les cellules contiennent 1 (présence) ou une valeur d’abondance.
  5. Calcul des indices taxonomiques
    • vegan::taxondive() est appliqué séparément pour chaque type de données.
  6. Export des résultats
    • Chaque événement (event_id) est associé aux six métriques pour les trois types de données.
    • Les résultats sont sauvegardés sous :
      • diversite_taxonomique.csv

5.11 Dissimilarité de Bray-Curtis (BC)

L’indice de Bray-Curtis est une mesure de dissimilarité qui quantifie la différence de composition spécifique entre deux échantillons en fonction de l’abondance ou de la biomasse des espèces. Cet indice permet d’évaluer dans quelle mesure des communautés biologiques diffèrent en termes de composition et de structure.

L’indicateur est défini comme suit :

\[BC_{ij} = 1 - \frac{2 \sum_{s=1}^{S} \min(X_{i,s}, X_{j,s})}{\sum_{s=1}^{S} (X_{i,s} + X_{j,s})}\]

où :

  • \(BC_{ij}\) est l’indice de dissimilarité entre les événements \(i\) et \(j\).
  • \(S\) est le nombre total d’espèces observées dans les deux échantillons.
  • \(X_{i,s}\) et \(X_{j,s}\) sont les abondances ou les biomasses de l’espèce \(s\) dans les événements \(i\) et \(j\).

L’indice varie entre 0 et 1 :

  • \(BC = 0\) → Les deux échantillons sont identiques en composition.
  • \(BC = 1\) → Les deux échantillons sont complètement différents.

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données d’abondance
    • Les données sont extraites de la table biodiversity de la base de données relationnelle SQLite.
    • Cinq sous-ensembles sont créés :
      • Données en n : Nombre total d’individus.
      • Données en n/m² : Nombre d’individus par unité de surface.
      • Données en g : Biomasse totale.
      • Données en n/m² : Biomasse totale par unité de surface.
      • Données d’occurrence
  2. Filtrage des données d’abondance (n/m²) et de biomasse (g/m²)
    • Seules les données où le type de mesure (measurement_type) est "abundance" et "biomass" ont été sélectionnées.
    • Deux ensembles distincts de données ont été générés :
      • Données en n/m² → Reflète la densité des individus.
      • Données en g/m² → Reflète la biomasse par unité de surface.
  3. Agrégation par événement et espèce
    • Pour chaque événement d’échantillonnage (event_id), la somme des abondances ou des biomasses par espèce (species_id) a été calculée séparément pour n/m² et g/m².
  4. Conversion en format matriciel
    • Les données ont été transformées en matrices communautaires où :
      • Les événements sont en lignes.
      • Les espèces sont en colonnes.
      • Les cellules contiennent l’abondance ou la biomasse totale par espèce.
    • Les matrices ont été transposées afin d’avoir les espèces en lignes et les événements en colonnes, conforme aux exigences de vegan::vegdist().
  5. Calcul de la matrice de dissimilarité Bray-Curtis
    • L’indice de dissimilarité entre tous les événements a été calculé séparément pour n/m² et g/m² avec vegan::vegdist(method = "bray").
    • Seule la partie inférieure de la matrice a été conservée pour éviter les doublons.
    • La dissimilarité des évènements d’échantillonnage où aucune espèce n’a été identifiée a été indiquée à 0.
  6. Export des résultats
    • Les valeurs calculées sont exportées sous format .csv :
      • dissimilarite_bray_curtis.csv

5.12 Richesse fonctionnelle (FR)

Statut

L’application de l’indice FR dépend de la disponibilité des traits fonctionnels pour les espèces présentes. Ces données n’ont pu être récoltées pour cette itération du projet.

L’indice de diversité fonctionnelle (FR) est une mesure de la richesse fonctionnelle d’une communauté biologique, c’est-à-dire l’étendue des traits fonctionnels présents au sein d’un échantillon (Mason et al., 2005). Contrairement aux indices de diversité taxonomique qui se basent sur le nombre d’espèces, cet indicateur repose sur les différences fonctionnelles entre les espèces, offrant ainsi une vision plus écologique de la biodiversité.

La richesse fonctionnelle est généralement définie comme le volume occupé par l’ensemble des espèces dans un espace multi-dimensionnel de traits. Un écosystème présentant une grande diversité fonctionnelle peut être plus résilient et performant en termes de processus écologiques.

L’indice FR est calculé à partir des distances fonctionnelles entre les espèces d’un échantillon, définies par un ensemble de traits biologiques pertinents.

L’indice FR est défini comme suit :

\[ FR_i = \sum_{s=1}^{S} d(T_{i,s}) \tag{5.9}\]

où :

  • \(FR_i\) = Indice de diversité fonctionnelle pour l’événement \(i\),
  • \(S\) = Nombre total d’espèces dans l’échantillon,
  • \(d(T_{i,s})\) = Distance fonctionnelle entre les espèces en fonction de leurs traits \(T\).

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données fonctionnelles
    • Les données sont extraites de la table biodiversity et enrichies avec des traits fonctionnels.
    • Les espèces sans traits disponibles sont exclues du calcul.
  2. Construction de la matrice de traits fonctionnels
    • Une matrice espèces-traits est créée avec les valeurs standardisées de chaque trait pertinent.
  3. Calcul des distances fonctionnelles
    • Une métrique de distance (ex. Gower, Euclidienne) est appliquée sur la matrice des traits.
    • La méthode de convex hull ou de dispersion des traits peut être utilisée pour estimer la diversité fonctionnelle.
  4. Agrégation par événement
    • Pour chaque événement d’échantillonnage (event_id), la richesse fonctionnelle est calculée.

5.13 Équitabilité fonctionnelle (FE)

Statut

L’application de l’indice FE dépend de la disponibilité des traits fonctionnels pour les espèces présentes. Ces données n’ont pu être récoltées pour cette itération du projet.

L’indice de diversité fonctionnelle (FE) est une mesure de l’uniformité fonctionnelle d’une communauté biologique, c’est-à-dire la manière dont les traits fonctionnels sont répartis entre les espèces d’un échantillon donné (Mason et al., 2005). Cet indicateur permet d’évaluer si certaines espèces ou groupes fonctionnels dominent l’écosystème ou si les traits sont uniformément distribués.

Une distribution plus homogène des traits fonctionnels peut favoriser la stabilité des écosystèmes en assurant un partage équilibré des ressources et des niches écologiques. À l’inverse, une faible équitabilité fonctionnelle pourrait indiquer une spécialisation accrue ou un environnement sous pression avec certaines espèces prédominantes.

L’indice FE est défini comme suit :

\[ FE_i = \frac{H'_{func}}{\log(S)} \tag{5.10}\]

où :

  • \(FE_i\) = Indice d’équitabilité fonctionnelle pour l’événement \(i\),
  • \(H'_{func}\) = Indice de Shannon appliqué aux distances fonctionnelles entre espèces,
  • \(S\) = Nombre total d’espèces dans l’échantillon.

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données fonctionnelles
    • Les données sont extraites de la table biodiversity et enrichies avec des traits fonctionnels.
    • Les espèces sans traits disponibles sont exclues du calcul.
  2. Construction de la matrice de traits fonctionnels
    • Une matrice espèces-traits est créée avec les valeurs standardisées de chaque trait pertinent.
  3. Calcul des distances fonctionnelles et de l’équitabilité
    • Une métrique de distance (ex. Gower, Euclidienne) est appliquée sur la matrice des traits.
    • L’indice de Shannon est calculé sur la distribution des distances fonctionnelles entre espèces.
    • La valeur est normalisée par le logarithme du nombre total d’espèces.
  4. Agrégation par événement
    • Pour chaque événement d’échantillonnage (event_id), l’équitabilité fonctionnelle est calculée.

5.14 Divergence fonctionnelle (FD)

Statut

L’application de l’indice FD dépend de la disponibilité des traits fonctionnels pour les espèces présentes. Ces données n’ont pu être récoltées pour cette itération du projet.

L’indice de diversité fonctionnelle (FD) est une mesure de la divergence fonctionnelle, c’est-à-dire la manière dont les espèces sont réparties dans l’espace des traits fonctionnels (Mason et al., 2005). Il permet d’évaluer si les espèces occupent des positions extrêmes dans l’espace fonctionnel ou si elles sont concentrées autour d’une moyenne.

Une forte divergence fonctionnelle peut indiquer une spécialisation accrue des espèces et une exploitation optimisée des ressources écologiques, tandis qu’une faible divergence peut révéler une redondance fonctionnelle élevée, où plusieurs espèces remplissent des rôles similaires dans l’écosystème.

L’indice FD est défini comme suit :

\[ FD_i = \frac{\sum_{s=1}^{S} w_s \cdot d(T_{i,s}, \bar{T_i})}{\sum_{s=1}^{S} w_s} \tag{5.11}\]

où :

  • \(FD_i\) = Indice de divergence fonctionnelle pour l’événement \(i\),
  • \(S\) = Nombre total d’espèces dans l’échantillon,
  • \(w_s\) = Abondance relative de l’espèce \(s\),
  • \(d(T_{i,s}, \bar{T_i})\) = Distance fonctionnelle de l’espèce \(s\) par rapport au centre fonctionnel moyen \(\bar{T_i}\).

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données fonctionnelles
    • Extraction des données à partir de la table biodiversity et enrichissement avec des traits fonctionnels.
    • Exclusion des espèces sans traits disponibles.
  2. Construction de la matrice de traits fonctionnels
    • Création d’une matrice espèces-traits standardisée.
  3. Calcul des distances fonctionnelles
    • Utilisation d’une métrique de distance (ex. Gower, Euclidienne) pour évaluer la proximité des espèces dans l’espace fonctionnel.
    • Détermination du centre fonctionnel moyen \(\bar{T_i}\).
  4. Calcul de la divergence fonctionnelle
    • Pondération des distances fonctionnelles par l’abondance relative des espèces.
    • Normalisation par la somme des abondances relatives.
  5. Agrégation par événement
    • Pour chaque événement d’échantillonnage (event_id), la divergence fonctionnelle est calculée.

5.15 Bentix

Statut

L’application de l’indice Bentix nécessite une classification des espèces benthiques selon leur tolérance aux perturbations. Bien que le package R BBI (Cordier, 2018) permette le calcul de cet indice, une consolidation appropriée des données taxonomiques de ce projet seront nécessaires pour un calcul adéquat de cet indicateur.

L’indice Bentix est un indicateur biologique permettant d’évaluer la qualité écologique des environnements benthiques en fonction de la réponse des groupes d’espèces aux perturbations anthropiques (Simboura & Zenetos, 2002). Il repose sur la pondération des espèces benthiques en fonction de leur sensibilité ou de leur tolérance aux perturbations environnementales, notamment l’enrichissement organique et la pollution.

Les organismes benthiques sont classés en deux groupes écologiques :

  • Groupe tolérant : Espèces opportunistes résistantes aux perturbations (ex. annélides opportunistes).
  • Groupe sensible : Espèces sensibles aux perturbations qui disparaissent lorsque l’environnement se détériore.

L’indice Bentix est calculé selon la proportion d’espèces sensibles et tolérantes dans l’échantillon.

\[ BENTIX_i = \frac{\sum_{s \in S_{sens}} w_s \cdot A_s}{\sum_{s=1}^{S} A_s} + 2 \times \frac{\sum_{s \in S_{tol}} w_s \cdot A_s}{\sum_{s=1}^{S} A_s} \tag{5.12}\]

où :

  • \(BENTIX_i\) = Indice Bentix pour l’événement \(i\),
  • \(S\) = Nombre total d’espèces dans l’échantillon,
  • \(S_{sens}\) = Ensemble des espèces sensibles aux perturbations,
  • \(S_{tol}\) = Ensemble des espèces tolérantes aux perturbations,
  • \(A_s\) = Abondance de l’espèce \(s\),
  • \(w_s\) = Pondération de l’espèce \(s\) selon sa sensibilité.

Les étapes spécifiques du traitement des données sont les suivantes :

  1. Filtrage des données benthiques
    • Extraction des données de la table biodiversity pour les organismes benthiques.
    • Attribution d’une catégorie écologique (sensible/tolérant) à chaque espèce en utilisant une classification de référence.
  2. Calcul des abondances pondérées et calcul de l’indice Bentix à l’aide du package R BBI (Cordier, 2018).
    • Séparation des abondances en deux groupes (sensibles et tolérants).
    • Calcul des proportions relatives de chaque groupe.
    • Application de la formule de calcul en pondérant les abondances par leur tolérance.
    • Calcul de l’indice Bentix pour chaque événement d’échantillonnage (event_id).
  3. Export des résultats
    • Chaque événement (event_id) est associé aux six métriques pour les trois types de données.

5.16 Indice de Suivi du Benthos (ISBg/ISBm)

Statut

L’application de l’indice ISBg/ISBm nécessite une classification des espèces benthiques selon leur tolérance aux perturbations et leur réponse aux conditions environnementales spécifiques aux cours d’eau. Les données nécessaires à sont application n’ont pu etre récoltées pour cette itération du projet.

L’Indice de Suivi du Benthos (ISBg/ISBm) est un indice développé pour évaluer l’état écologique des cours d’eau en se basant sur la structure des communautés benthiques (Pelletier & Moisan, 2012). Il repose sur l’identification et la classification des taxons benthiques en fonction de leur sensibilité aux perturbations, notamment les impacts liés à l’eutrophisation, la pollution organique et la modification des habitats.

L’indice se décline sous deux variantes :

  • ISBg : Basé sur les groupes taxonomiques principaux des invertébrés benthiques.
  • ISBm : Basé sur des mesures plus détaillées, prenant en compte la diversité spécifique et des traits fonctionnels.

L’objectif principal de l’ISBg/ISBm est de caractériser l’impact des pressions environnementales en utilisant des indicateurs biologiques représentatifs de l’état de santé des cours d’eau.

L’indice ISBg/ISBm repose sur l’analyse de plusieurs métriques :

  1. La composition des communautés benthiques (présence/absence, abondance relative des groupes).
  2. La tolérance des taxons aux perturbations (classification en fonction de la sensibilité écologique).
  3. Les traits fonctionnels (stratégies d’alimentation, préférences d’habitat).
  4. Les conditions environnementales locales (qualité de l’eau, substrat, vitesse du courant).

L’indice ISBg/ISBm est calculé selon la formule :

\[ ISB_i = \sum_{s=1}^{S} w_s \cdot A_s \tag{5.13}\]

où :

  • \(ISB_i\) = Indice de Suivi du Benthos pour l’événement \(i\),
  • \(S\) = Nombre total de taxons benthiques identifiés,
  • \(A_s\) = Abondance de l’espèce \(s\),
  • \(w_s\) = Pondération de l’espèce \(s\) selon sa tolérance aux perturbations.

5.17 Benthic Opportunistic Annelida Amphipod (BO2A)

Caution

L’indice BO2A analysant la proportion relative d’annélides opportunistes et d’amphipodes. Cette classification devra être validée afin d’assurer une évaluation adéquate de cet indice.

L’indice BO2A (Benthic Opportunistic Annelida Amphipod Index) est un indicateur biologique permettant d’évaluer l’état écologique des environnements benthiques en analysant la proportion relative d’annélides opportunistes et d’amphipodes dans les communautés benthiques (Dauvin & Ruellet, 2009). Il est basé sur l’hypothèse que les annélides opportunistes sont plus tolérants aux perturbations, tandis que les amphipodes sont plus sensibles aux modifications environnementales, notamment la pollution organique et la désoxygénation.

L’indice BO2A repose sur l’analyse des proportions relatives des deux groupes taxonomiques suivants :

  • Annélides opportunistes (Tolérants aux perturbations) : annelides opportunistes (familles Spionidae et Capitellidae, classe Clittellata) qui prolifèrent en conditions enrichies en matière organique.
  • Amphipodes (Sensible aux perturbations) : amphipodes (famille Pontoporeiidae) dont l’abondance diminue en conditions dégradées.

L’indice BO2A est défini comme suit :

\[ BO2A_i = \log_2 \left( \frac{f_{p+o}}{f_a + 1} + 1 \right) \tag{5.14}\]

où :

  • \(BO2A_i\) = Indice BO2A pour l’événement \(i\),
  • \(f_{p+o}\) = Fréquence des annélides opportunistes (proportion des annélides opportunistes sur l’ensemble des individus présents dans l’échantillon),
  • \(f_a\) = Fréquence des amphipodes (proportion d’amphipodes sur l’ensemble des individus présents dans l’échantillon).

Le BO2A est basé sur une relation antagoniste entre amphipodes et annélides opportunistes :

  • BO2A proche de 0 → Faible proportion d’opportunistes, indiquant un bon état écologique.
  • BO2A proche de 1 → Forte dominance d’opportunistes, indiquant un état écologique perturbé.

5.20 Benthic Quality Index (BQI)

Statut

Le BQI nécessite une base de données spécifique aux espèces, intégrant des valeurs de tolérance, d’abondance et de diversité pour évaluer la qualité benthique. Ces données n’ont pu être récoltées pour cette itération du projet.

L’indice Benthic Quality Index (BQI) est un indicateur écologique utilisé pour évaluer l’état des communautés benthiques en combinant trois paramètres clés : la tolérance des espèces à la pollution, leur abondance et la diversité spécifique (Rosenberg et al., 2004). Cet indice permet d’obtenir une mesure intégrée de la qualité des habitats benthiques en fonction de la présence d’espèces sensibles et tolérantes aux perturbations environnementales.

Le BQI est calculé à partir de la formule suivante :

\[ BQI_i = \frac{\sum_{s=1}^{S} (A_s \cdot T_s)}{N_i} \cdot \log_2 (S_i + 1) \tag{5.17}\]

où :

  • \(BQI_i\) = Indice BQI pour l’échantillon \(i\),
  • \(A_s\) = Abondance de l’espèce \(s\) dans l’échantillon,
  • \(T_s\) = Score de tolérance de l’espèce \(s\) (plus faible pour les espèces sensibles, plus élevé pour les espèces tolérantes),
  • \(S_i\) = Nombre total d’espèces dans l’échantillon,
  • \(N_i\) = Nombre total d’individus dans l’échantillon.

L’ajout du logarithme de la richesse spécifique permet de pondérer l’indice en tenant compte de la diversité des communautés, offrant ainsi une meilleure évaluation de la résilience et de la stabilité écologique du milieu.

5.25 Multivariate AMBI (M-AMBI\(^*\))

Statut

L’application de l’indice M-AMBI* nécessite une classification des espèces benthiques selon leur tolérance aux perturbations, ainsi que des données sur la richesse spécifique et la diversité. Bien que le package R BBI (Cordier, 2018) permette le calcul de l’indice AMBI, le M-AMBI* intègre également la richesse spécifique et l’indice de Shannon-Wiener, nécessitant un traitement supplémentaire via vegan (Oksanen et al., 2024) et une normalisation Min-Max. De plus, une consolidation appropriée des données taxonomiques de ce projet sera nécessaire pour un calcul adéquat de cet indicateur.

L’indice Multivariate AZTI Marine Biotic Index simplifié (M-AMBI*) est une version améliorée de l’AMBI (Borja et al., 2000), intégrant des métriques complémentaires pour fournir une évaluation robuste de l’état écologique des habitats benthiques (Sigovini et al., 2013). Il repose sur trois paramètres :

  1. L’AMBI, qui classe les espèces selon leur tolérance aux perturbations.
  2. La richesse spécifique (S), qui reflète la diversité taxonomique.
  3. L’indice de Shannon-Wiener (H’), qui mesure la répartition des individus entre les espèces.

Méthodologie

Contrairement à la version originale de M-AMBI, qui utilisait une analyse en composantes principales (PCA), les travaux de Sigovini et al. (2013) ont démontré que M-AMBI* fournit des résultats équivalents en utilisant une simple moyenne des valeurs normalisées.

L’indice M-AMBI* est donc calculé comme suit :

\[ M\text{-}AMBI^*_i = \frac{S'_i + H'_i - AMBI'_i}{3} \]

où :

  • \(S'_i\), \(H'_i\) et \(AMBI'_i\) sont Min-Max normalisés sur la plage [0,1].
  • Cette approche simplifie l’algorithme sans altérer la fiabilité des résultats.

Étapes spécifiques du traitement des données

  1. Filtrage des données benthiques
    • Extraction des données de la table biodiversity pour les organismes benthiques.
  2. Calcul des métriques écologiques
    • AMBI est calculé avec le package BBI.
    • Richesse spécifique (S) est calculée avec vegan::specnumber().
    • Indice de Shannon-Wiener (H’) est estimé avec vegan::diversity().
  3. Application de la normalisation et agrégation des valeurs
    • Min-Max Normalisation des indices AMBI, S et H’ entre 0 et 1 : \[ X'_i = \frac{X_i - X_{\text{min}}}{X_{\text{max}} - X_{\text{min}}} \]
    • Agrégation des trois métriques par moyenne pondérée pour obtenir M-AMBI*.
  4. Export des résultats
    • Chaque événement (event_id) est associé aux valeurs calculées.
    • Les résultats sont sauvegardés sous trois fichiers :
      • bbi.csv
      • mambi_indices.csv

5.26 Infaunal Trophic Index (ITI)

Statut

L’application de l’indice ITI nécessite une classification des espèces benthiques selon leur mode d’alimentation et leur réponse aux perturbations. L’Infaunal Trophic Index (ITI) est un indice écologique qui évalue la structure trophique des communautés benthiques en fonction de la proportion d’espèces appartenant à différents groupes fonctionnels (Word, 1979). Cet indice est principalement utilisé pour analyser l’effet des perturbations anthropiques, notamment l’enrichissement organique des sédiments, sur les écosystèmes marins.

Les espèces benthiques sont classées en quatre groupes trophiques en fonction de leur mode d’alimentation et de leur tolérance aux perturbations :

  • Groupe 1 : Espèces suspensivores filtrantes (sensitive aux perturbations).
  • Groupe 2 : Espèces suspensivores et dépositivores non sélectives.
  • Groupe 3 : Espèces dépositivores sélectives.
  • Groupe 4 : Espèces opportunistes (tolérantes aux conditions anoxiques et enrichies en matière organique).

L’indice ITI est basé sur la proportion d’espèces appartenant à ces groupes et leur abondance relative dans l’échantillon. L’ITI est calculé comme suit :

\[ ITI_i = \frac{\sum_{s=1}^{S} w_s \cdot A_s}{\sum_{s=1}^{S} A_s} \tag{5.22}\]

où :

  • \(ITI_i\) = Indice ITI pour l’événement \(i\),
  • \(S\) = Nombre total d’espèces dans l’échantillon,
  • \(A_s\) = Abondance de l’espèce \(s\),
  • \(w_s\) = Pondération de l’espèce \(s\) selon son groupe trophique.

Étapes spécifiques du traitement des données

  1. Filtrage des données benthiques
    • Extraction des données de la table biodiversity pour les organismes benthiques.
  2. Calcul des abondances pondérées et application de l’indice ITI avec BBI
    • Attribution d’une catégorie trophique (Groupe 1 à 4) à chaque espèce en utilisant une classification de référence.

5.27 Indicator Species Index (ISI)

Statut

L’application de l’indice ISI repose sur l’identification d’espèces indicatrices à l’aide d’études statistiques basées sur la méthode de raréfaction de Hurlbert. Bien que le package R BBI (Cordier, 2018) permette le calcul de cet indice, une consolidation appropriée des données taxonomiques et écologiques de ce projet sera nécessaire pour une estimation adéquate de cet indicateur.

L’Indicator Species Index (ISI) est un indice conçu pour identifier des espèces indicatrices en fonction de leur présence et de leur abondance dans différents niveaux de qualité environnementale (Rygg, 2002). L’ISI s’appuie sur une méthode de raréfaction, dérivée de l’approche développée par Hurlbert (1971), afin d’évaluer la diversité et la spécificité des espèces dans différents types d’habitats. L’objectif principal de l’ISI est d’identifier des espèces qui réagissent de manière caractéristique aux conditions environnementales, permettant ainsi de classer les écosystèmes en fonction de leur état écologique.

L’indice ISI est basé sur la relation entre :

  1. L’occurrence des espèces indicatrices, qui sont fortement associées à un état écologique particulier.
  2. L’abondance relative, qui reflète leur importance dans la communauté.
  3. L’approche de raréfaction de Hurlbert, qui standardise les comparaisons entre échantillons en tenant compte des différences d’effort d’échantillonnage.

L’indice ISI est défini par :

\[ ISI_i = \sum_{s=1}^{S} w_s \cdot A_s \tag{5.23}\]

où :

  • \(ISI_i\) = Indice ISI pour l’événement \(i\),
  • \(S\) = Nombre total d’espèces dans l’échantillon,
  • \(A_s\) = Abondance de l’espèce \(s\),
  • \(w_s\) = Poids de l’espèce \(s\) basé sur son pouvoir indicateur.

Les poids \(w_s\) sont attribués en fonction d’une classification statistique des espèces indicatrices, obtenue par une analyse de raréfaction.

Étapes spécifiques du traitement des données

  1. Identification des espèces indicatrices
    • Extraction des données taxonomiques et écologiques.
    • Sélection des espèces présentant des occurrences significatives dans des états écologiques spécifiques.
  2. Application de la méthode de raréfaction de Hurlbert
    • Estimation du nombre d’espèces attendues par échantillon standardisé.
    • Normalisation des abondances pour éviter les biais liés à l’effort d’échantillonnage.
  3. Calcul de l’ISI avec BBI
    • Attribution d’un score indicateur basé sur des classifications de référence.
    • Agrégation des espèces indicatrices et pondération par leurs abondances relatives.
    • Application de la formule ISI pour chaque événement d’échantillonnage (event_id).
  4. Export des résultats
    • Chaque événement (event_id) est associé aux valeurs calculées.
    • Les résultats sont sauvegardés sous trois fichiers :
      • bbi.csv
